# Azure Governance

Azure governance maps existing local evidence controls to target-state cloud controls.

- Manifests and checksums map to lineage and data quality evidence.
- Synthetic-data declarations map to data classification and onboarding controls.
- Model cards, metric reports, and guardrails map to responsible AI governance.
- Human-review flags map to operational approval workflows.
- Diagnostic settings map to Log Analytics and audit evidence.
- Purview collections map to environment, data-zone, and domain boundaries.

This governance layer is documentation only. The final portfolio evidence pack and readiness
validator cross-link this boundary so reviewers can distinguish implemented local evidence from
Azure target-state design.
