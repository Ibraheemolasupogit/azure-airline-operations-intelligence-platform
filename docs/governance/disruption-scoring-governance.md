# Disruption Scoring Governance

Milestone 7 disruption scoring is governed as transparent synthetic evidence.

## Controls

- Every run requires an explicit Milestone 3 validation report directory.
- Required processed files are reconciled to the validation manifest.
- Optional upstream analytics are accepted only when they derive from the same validation run.
- Forward-looking risk excludes actual delay, cancellation, diversion, and reactionary outcomes.
- Retrospective severity labels actual synthetic outcomes as historical context.
- Alerts are capped, deterministic, conservative, and require human review.

## Out Of Scope

No optimisation, passenger rebooking, crew or aircraft reassignment, GenAI summaries, dashboards,
live monitoring, Azure deployment, or operational automation is implemented.
